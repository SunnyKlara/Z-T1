# 🔧 RideWind T1 涡轮风扇 — UX出厂测试大纲

> **文档目的**：模拟嵌入式产品出厂前的全功能用户体验测试。
> 覆盖硬件端所有可见、可感知、可交互的功能场景。
> 每个测试用例 = 操作步骤 + 预期结果 + 实际结果（空）。

---

## 一、产品概况

### 1.1 硬件平台
```
MCU:      ESP32-S3
显示屏:   GC9A01 240×240 圆形 LCD (SPI, IO4/5/6/7)
编码器:   EC11 旋转编码器 (A=IO17, B=IO18, 按键=IO8 低电平有效)
LED灯带:  WS2812B × 13颗
          - 物理灯带1 (IO41, 10颗)：左2 + 中6 + 右2
          - 物理灯带2 (IO16, 3颗)：尾3
风扇:     LEDC PWM (IO10)，占空比 0-100%
空心杯:   LEDC PWM (IO14)，25kHz，占空比 0-100%
加湿器:   GPIO 开/关 (IO40)
音频:     MAX98357 I2S (IO11/12/13)，44100Hz 16-bit 立体声
BLE:      Service 0xFFE0, Char 0xFFE1, 设备名 "T1"
WiFi:     STA模式, TCP server :8080
存储:     NVS + LittleFS (2MB Flash分区)
```

### 1.2 软件架构速览
```
app_main() [Core 0]
  ├── 初始化：NVS → AppState → 外设(LCD/LED/PWM/GPIO/Audio)
  ├── 启动Logo 3秒 → 进菜单UI5
  └── main_task() [Core 1, 20ms循环]
        ├── 处理BLE命令队列
        ├── ui_manager_update()  →  UI状态机 + 界面渲染
        └── led_effects_process() → LED特效(渐变/流光/呼吸)

BLE回调 [Core 0]
  └── process_rx_data() → 协议解析 → xQueueSend → main_task处理

WiFi音频 [TCP Server Task, Core 0]
  └── 接收TCP PCM → audio_engine_feed → 环形缓冲区 → audio_output_task [Core 1]

引擎合成 [FreeRTOS Task]
  └── 程序化合成 → I2S DMA 输出
```

### 1.3 UI界面一览（6个活跃界面）
```
UI1  Speed    — 速度控制 + 油门模式（单击进入/退出，长按加减速）
UI2  Color    — 14色预设 + 流光效果
UI3  RGB      — 4灯带×3通道自定义调色
UI4  Bright   — 亮度 + 呼吸效果
UI5  Menu     — 导航菜单（6页滑动动画）
UI6  Logo     — Logo图片管理
UI8  Treadmill — 跑步机Forza表盘
```
> UI7（音量）已禁用，任何路径访问都被重定向到菜单。

### 1.4 编码器事件一览
| 事件 | 触发条件 | 超时 |
|------|---------|------|
| `ENC_EVT_ROTATE` | 旋转，累积delta ±20 | 3ms消抖 |
| `ENC_EVT_CLICK` | 单击 | 400ms超时 |
| `ENC_EVT_DOUBLE_CLICK` | 双击 | 400ms超时 |
| `ENC_EVT_TRIPLE_CLICK` | 三连击 | 400ms超时 |
| `ENC_EVT_LONG_PRESS` | 按住≥800ms | — |
| `ENC_EVT_PRESS` / `ENC_EVT_RELEASE` | 按下/释放边沿 | — |

## 二、UI1 — 速度控制 / 风扇

**界面元素**：F4风格背景 + 风速表图标 + 大号速度数字(右对齐) + kmh/mph单位标签 + 加湿器LED指示灯

---

### 2.1 基础速度调节

| ID | 测试场景 | 操作步骤 | 预期结果 |
|----|---------|---------|---------|
| S1-01 | 旋转调速度 | 开机→菜单→旋转到Speed→单击进入。旋转编码器 | 速度数字从0递增/递减，0–340(kmh)或0–211(mph)。风扇转速同步变化。引擎声随即启动(RPM跟随速度)。BLE发送`SPEED_REPORT:display:unit` |
| S1-02 | 速度上限 | 旋转到340kmh(或211mph)后继续旋 | 数字不再增大，卡在最大值 |
| S1-03 | 速度归零 | 旋转到0 | 数字归0。风扇停转。引擎声停止。BLE发送`SPEED_REPORT:0:0` |
| S1-04 | 数字显示一致性 | 旋转到10、20、30...100kmh | 显示值 = 内部值 × 3.4（kmh），无跳变。如内部值=10，显示=34 |

### 2.2 单位切换 (kmh ↔ mph) — 仅APP端

> **R4变更**：硬件端移除单击切单位。单位切换仅通过 BLE `UNIT:x` 命令（APP端控制）。
> 硬件端单击改为进入/退出油门模式（见2.4）。

| ID | 测试场景 | 操作步骤 | 预期结果 |
|----|---------|---------|---------|
| S1-05 | APP切单位 | APP发送`UNIT:1` | 单位标签变为mph，数字换算正确。BLE回复`OK:UNIT` |
| S1-06 | APP切回 | APP发送`UNIT:0` | 单位切回kmh |
| S1-07 | 单位持久化 | 切mph→双击回菜单→重启设备→进Speed | 单位应保持mph（NVS持久化） |

### 2.3 加湿器控制

| ID | 测试场景 | 操作步骤 | 预期结果 |

| S1-09 | 长按开关加湿器 | 长按编码器(≥800ms) | 加湿器切换开/关。LED指示灯：熄灭→绿灯，绿灯→熄灭。BLE发送`WUHUAQI_REPORT:0/1` |
| S1-10 | 加湿器状态指示 | 观察LED指示灯 | 关=红/灭, 开=绿, 油门=橙。与硬件GPIO状态一致 |

### 2.4 油门模式（单击进入/退出 — R4变更）

> **R4变更**：单击进入油门模式，再单击退出。替代旧的三连击。
> 油门模式：单击进入 → 长按加速 → 松开减速 → 单击或旋转退出（保持当前速度）。

| ID | 测试场景 | 操作步骤 | 预期结果 |
|----|---------|---------|---------|
| S1-12 | 进入油门模式 | Speed界面→单击编码器 | 加湿器强制打开。LED变橙色(油门指示)。BLE发送`THROTTLE_REPORT:1`。引擎声启动(若未启动)。速度保持当前值 |
| S1-13 | 单击退出油门 | 油门模式中→单击 | 退出油门。保持当前速度（不归零）。LED恢复之前状态。BLE发送`THROTTLE_REPORT:0` |
| S1-14 | 旋转退出油门 | 油门中旋转编码器 | 退出油门。保持当前速度。加湿器恢复之前状态 |
| S1-15 | 长按加速 | 油门中长按编码器不松手 | 速度从当前值逐渐上升（每200ms+1），风扇PWM同步增大，引擎RPM升高 |
| S1-16 | 加速到顶 | 持续长按直到速度=100 | 速度到达100后不再上升 |
| S1-17 | 松开减速 | 加速中松开按钮 | 速度每200ms-1递减。松开后缓减速，再按重新加速 |
| S1-18 | 减速到底自动退出 | 松手后速度降到0 | 自动退出油门模式。恢复之前加湿器状态。BLE发送`THROTTLE_REPORT:0` |
| S1-19 | BLE THROTTLE:1远程进入 | APP发送`THROTTLE:1` | 油门模式激活。加湿器打开 |
| S1-20 | BLE THROTTLE:0远程退出 | APP发送`THROTTLE:0` | 退出油门。速度和风扇归零。加湿器恢复 |
| S1-21 | 油门中BLE发FAN命令 | APP发送`FAN:80` | 收到`ERR:FAN:THROTTLE`，风扇不响应 |
| S1-22 | 油门中BLE发SPEED:0 | APP发送`SPEED:0` | 引擎停止。编码器队列被清空防止立即重启 |

### 2.5 引擎声
