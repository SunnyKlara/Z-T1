## 一、一句话理解产品

**RideWind** 是一款 **智能风洞设备** 的配套 APP + 固件。

用户通过 APP 控制风洞设备，实现：
- 风速调节（0-100%）
- LED 灯光效果（14 种预设颜色）
- 自定义 Logo 显示（LCD 屏幕）
- 引擎音频模拟（I2S 输出）

**核心价值**：让用户在室内也能体验真实风洞的感觉（飞行、跳伞、风洞训练）。

## 二、技术架构（30 秒理解）

```
┌─────────────────────────────────────────────────────────┐
│                      用户手机 (APP)                      │
│  Flutter (Dart)                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  主页    │  │ 设置面板 │  │ Logo上传 │              │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘              │
│       └──────────────┴──────────────┘                   │
│                     BLE 通信                            │
└─────────────────────────┬───────────────────────────────┘
                          │ BLE (ESP32)
┌─────────────────────────┴───────────────────────────────┐
│                    风洞设备 (固件)                        │
│  ESP32-S3 (C)                                           │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ 电机控制 │  │ LED 控制 │  │ LCD 显示 │              │
│  └──────────┘  └──────────┘  └──────────┘              │
│  ┌──────────┐  ┌──────────┐                             │
│  │ 音频输出 │  │ 编码器   │                             │
│  └──────────┘  └──────────┘                             │
└─────────────────────────────────────────────────────────┘
```

**关键硬件**：
- 主控：ESP32-S3
- 电机：PWM 控制（引脚 GPIO18）
- LED：WS2812B × 6 颗（引脚 GPIO17）
- LCD：GC9A01 圆形屏 240×240（SPI）
- 音频：I2S 输出（DAC）
- 编码器：EC11 旋钮（GPIO4/5）
- 按键：BOOT 键（GPIO0）

## 五、踩过的坑（必读）

### 🔴 致命坑（必须避免）

1. **BLE MTU 分片**：长命令必须分片传输，固件端重组
2. **BLE 通知拥塞**：发送通知必须重试，处理 `ESP_GATT_CONGESTED`
3. **Logo 数据包乱序**：必须有序号校验，乱序包丢弃
4. **PSRAM 不足**：大缓冲区必须用 `heap_caps_malloc(..., MALLOC_CAP_SPIRAM)`
5. **CRC32 不一致**：两端必须使用相同多项式 `0xEDB88320`

### 🟡 严重坑（需要注意）

6. **I2S 音频爆音**：必须使用双缓冲，缓冲区 ≥ 2048 采样
7. **WS2812B 时序**：必须用 RMT 或 SPI，不能用软件延时
8. **GC9A01 初始化**：必须用完整序列，SPI ≤ 40MHz
9. **EC11 抖动**：必须软件消抖（5ms 延时）
10. **连接断开状态**：必须清理状态，通知 APP

> 完整坑位清单见 `known-pitfalls.md`

## 六、协议契约（核心）

### BLE 命令格式

```
命令: COMMAND_NAME:param1:param2\n
响应: OK:COMMAND_NAME:result\n
      ERROR:COMMAND_NAME:reason\n
```

### 核心命令

| 命令 | 说明 | 状态 |
|------|------|------|
| `SET_SPEED:value` | 设置风速 0-100 | ✅ |
| `SET_LED:index` | 设置 LED 颜色 0-13 | ✅ |
| `GET_STATUS` | 获取设备状态 | ✅ |
| `LOGO_UPLOAD:seq/total:crc:data` | 上传 Logo 数据包 | 🚧 |
| `SET_AUDIO:on/off` | 开关音频 | 🚧 |
