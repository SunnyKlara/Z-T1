# DR-002: 音频引擎方案

| 字段 | 值 |
|------|-----|
| **状态** | 已确认架构方向，素材方案待定 |
| **决策日期** | 2026-05-09 |
| **涉及模块** | 固件端 |
| **决策者** | Klara |

## 1. 背景

RideWind 使用纯程序化合成 (`engine_synth_procedural.c`) 生成引擎声音：锯齿波 + 低通滤波模拟引擎转速变化。问题是：
1. 音质上限低——怎么调参都像"电子游戏音效"
2. CPU 占用 40-60%，和 BLE/LCD/LED 抢资源
3. 声学工程调参极为困难，且结果不可预测
4. 无法达到目标参考水准（地平线级别的引擎音频体验）

## 2. 方案对比

| 方案 | 描述 | 音质 | CPU | 存储 | 开发周期 | 评估 |
|------|------|------|-----|------|---------|------|
| A: 实时合成 | 程序化生成波形 | ⭐⭐ | 40-60% | 0 | 2-4周 | ❌ 音质达不到 |
| B: MP3 样本 + I2S | 预录制 MP3, 按速度切换 | ⭐⭐⭐⭐ | 5-10% | ~2MB | 3-5天 | ✅ 选用 |
| C: 混合 | 合成底 + 样本叠加 | ⭐⭐⭐ | 15-25% | ~500KB | 2-3周 | 可后期补充 |
| D: 样本+合成辅助 | MP3主体 + 合成效果层 | ⭐⭐⭐⭐⭐ | 8-15% | ~2MB | 1-2周 | ✅ 目标方案 |

## 3. 最终选择

**方案 D: MP3 样本主导 + 程序化合成辅助**。

**架构分层**：

```
Layer 1: 样本引擎（主体音频）
  - 4-6 个 RPM 区间的 MP3 样本（idle / low / mid / high / redline / start / stop）
  - 按速度切换 + 交叉淡入淡出（50ms 过渡）
  - 使用 ESP-ADF esp_audio 组件解码 MP3

Layer 2: 合成效果层（辅助，极简）
  - 保留 RideWind 已有的短时效果（换挡 150ms 扫频、泄压阀白噪声、涡轮哨声）
  - 这些效果 CPU 占用极小（< 5%）

Layer 3: I2S 输出
  - MAX98357, 44.1kHz 16-bit 单声道
  - 双缓冲 2048 采样（防欠载爆音）
```

**RideWind 保留的部分**：
- `audio_engine` 总控架构（pause/resume/volume）
- `engine_synth_procedural` 的换挡音、泄压阀音效
- `audio_player` 的速度→音频映射逻辑
- I2S 双缓冲方案

**RideWind 替换的部分**：
- 锯齿波合成 → MP3 样本播放
- CPU 占用从 40-60% → 8-15%

## 4. 素材获取方案

**目标标准**: 地平线级别的引擎音频体验。

| 路径 | 内容 | 价格 | 质量 | 推荐 |
|------|------|------|------|------|
| Boom Library - Engine Sounds | 真实引擎录音，多 RPM 分层 | ~$200 | 顶级 | ✅ |
| Pro Sound Effects - Vehicle | 专业车辆音效 | ~$150 | 顶级 | ✅ |
| Freesound.org (CC0) | 用户上传，需筛选 | 免费 | 参差不齐 | 验证架构用 |
| AI 生成 (AudioCraft) | 文本→音频 | $0 | 不确定 | ❌ 不推荐 |

**素材规格要求**：
- 格式: WAV 44.1kHz 16-bit 单声道
- 内容: 同一引擎的 4-6 个 RPM 区间录音
- 每个录音 10-30 秒
- 必须包含启动/停止瞬态

## 5. 实施步骤

```
Phase 1: 架构验证（用免费 CC0 素材）
  1. ESP-ADF esp_audio 组件集成
  2. MP3/WAV 解码 → I2S 双缓冲输出
  3. 速度→样本切换 + 交叉淡入淡出
  4. 保留的合成效果层集成
  5. 验证: CPU < 15%, 无爆音, 切换无卡顿

Phase 2: 素材升级
  1. 购买商业音效库
  2. 裁剪/格式转换 → 适配 RPM 区间
  3. 替换素材 → 内测体验 → 调优过渡

Phase 3: 精细调优
  1. 根据实际体验调整淡入淡出参数
  2. 优化样本循环的无缝衔接
  3. 最终验收
```

## 6. 风险与对策

| 风险 | 对策 |
|------|------|
| MP3 解码 CPU 超标 | 用 WAV 原始格式（无解码开销），ESP32-S3 PSRAM 够大 |
| 样本循环有接缝 | 录音时录制完整循环（30秒以上），一般听不出 |
| 商业素材版权问题 | 购买时确认商用授权 |
| I2S 缓冲区欠载 | 双缓冲 + 增大缓冲（4096 采样） |
