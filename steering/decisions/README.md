# 技术决策记录 (Decision Records)

> **用途**: 记录每个关键技术方案的**最终决策、选择理由、替代方案、实施步骤**。
> **为什么存在**: 防止每次开新对话时重新讨论同样的技术问题、重新从旧代码"考古"。
>            新 AI 读完一篇 Decision Record = 继承了整段讨论的历史。

---

## 决策记录模板

每篇 Decision Record 必须包含以下章节：

```
# DR-XXX: 决策标题

| 字段 | 值 |
|------|-----|
| **状态** | 提议 / 已确认 / 已实施 / 已废弃 |
| **决策日期** | YYYY-MM-DD |
| **涉及模块** | APP端 / 固件端 / 两端 |
| **决策者** | (你的名字/角色) |

## 1. 背景

为什么需要做这个决策？当前有什么问题？

## 2. 方案对比

| 方案 | 优点 | 缺点 | 评估 |
|------|------|------|------|
| A   | ...  | ...  | ...  |
| B   | ...  | ...  | ...  |

## 3. 最终选择

**选方案 X**，理由：
- 理由1
- 理由2

## 4. 放弃的方案

明确写出**为什么不用**替代方案，避免后续 AI 重新提议。

## 5. 实施步骤

1. Step 1
2. Step 2
3. ...

## 6. 风险与对策

| 风险 | 对策 |
|------|------|
| ...  | ...  |

## 7. 相关参考

- 设计规范: `steering/specs/xxx.md`
- 协议契约: `steering/specs/protocol-contract.md`
- 旧项目参考: `reference/xxx`
```

## 使用规则

1. **决策确认后必须写 DR**，不能只靠对话记录
2. **写代码前先查 DR**，确认方案已经决策过
3. **如果要推翻已有决策** → 必须新的 DR 说明推翻理由 + 新的替代方案
4. **未决策的功能不能开始实现** → 先讨论 → 写 DR → 确认 → 实施
