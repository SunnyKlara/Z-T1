## 第二圈: 通信联通 (Week 3-4)

目标: APP发命令 -> 设备响应 -> 真实数据流动。

进入前需确认: DR-004(BLE连接体验方案)

### APP 端

| 天 | 任务 | 产出 |
|----|------|------|
| D1 | ble_scanner | BLE扫描实现(按0xFFE0过滤) |
| D2 | ble_connection | BLE连接/断开/MTU协商 |
| D3 | ble_read + ble_write | BLE收发 + 发送队列 |
| D4 | protocol_parser | 文本协议解析(纯函数) |
| D5 | command_builder | 命令构造(字符串拼接) |
| D6 | response_router | 响应路由(分发到Provider) |
| D7 | 第一轮联调 | APP发PING -> 设备回PONG |

### 固件端

| 天 | 任务 | 产出 |
|----|------|------|
| D1 | BLE GATT服务 | 广播T1, Service 0xFFE0 |
| D2 | 协议解析 | 文本协议解析 + 命令队列 |
| D3 | 命令分发 | 解析后路由到对应模块 |
| D4-D7 | APP+固件联调 | FAN/LED/WUHUA全命令走通 |

### 验收

- APP 扫描 -> 找到T1 -> 点击连接 -> 1秒内成功
- APP 拖动速度滑块 -> 风扇实时变化
- 断开 -> 设备重新广播

## 第三圈: 功能完善 (Week 5-12)

目标: 全部功能逐个实现、逐个验证。

进入前需确认: DR-005(Logo上传)/DR-006(LED特效)/DR-007(编码器交互)

### Week 5-6: LED + 风扇 + 加湿器

LED 14种预设/流水灯/亮度 + 风扇全模式(巡航/极速/展览/油门) + 加湿器

### Week 7-8: 编码器 + LCD

编码器交互(调速/调色/菜单) + LCD 7个页面(Speed/Color/RGB/Bright/Menu/Logo/Treadmill)

### Week 9-10: Logo + 音频

Logo上传(多槽位/删除) + 音频引擎(MP3样本播放/速度切换) + 效果层(换挡/泄压阀/涡轮)

### Week 11-12: WiFi + 数据绑定

WiFi配网/音频投射 + 4个Panel接入真实数据
