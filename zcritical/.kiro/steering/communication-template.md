# AI 沟通模板

> **职责**: 解决"怎么跟 AI 沟通"这个最折磨人的问题。不需要每次都想怎么说，照着模板填就行。
> **设计意图**: RideWind 开发中最大的痛苦不是写代码，是沟通——不知道怎么描述问题、不知道 AI 为什么不懂、来回拉扯耗尽耐心。此模板消除沟通摩擦。
> **用法**: 每次和新 AI 对话，先发 `START-HERE.md` 内容，然后套用下面的模板。

---

## 模板一：提新需求

```
【新功能需求】

功能名称: [一句话]

用户场景: 什么时候、什么情况下、用户想做什么？
  [用日常语言描述，不用技术术语]

操作路径: 从开始到结束的步骤
  1. [第一步]
  2. [第二步]
  3. ...

成功标准: 什么算是"做好了"？

AI 请回答:
  1. 先分析这个需求：合理吗？有我没考虑到的边界情况吗？
  2. 输出方案（不写代码）：涉及哪些文件？放在哪层？依赖谁？
  3. 提出你的专业建议：有没有更好的做法？有什么隐患？
```

### 示例（填好的）

```
【新功能需求】

功能名称: Logo 上传

用户场景: 在桌前把玩风洞模型时，想把车队 Logo 传到设备屏幕上

操作路径:
  1. 点击首页"定制"按钮
  2. 选择要上传的图片
  3. 看到 240×240 预览效果
  4. 确认上传
  5. 看到上传进度
  6. 完成后看到效果

成功标准:
  - 从选择到完成不超过 2 分钟
  - 任何步骤都知道发生了什么
  - 失败了知道为什么 + 可以重试

AI 请回答: 1.需求分析 + 2.方案 + 3.专业建议
```

## 模板二：反馈问题（不对）

```
【反馈问题】

文件/功能: [哪个功能出问题了]

实际现象: 
  [不是"它不对"，而是具体的："点了按钮没反应" / "数字跳回0"]

期望行为:
  [你希望它怎么样]

复现步骤:
  1. [第一步]
  2. [第二步]
  3. [看到问题]

AI 请回答:
  1. 分析可能原因，列出 2-3 个最可能的
  2. 输出修复方案
  3. 会影响其他功能吗？
```

## 模板三：讨论设计（不确定怎么做）

```
【讨论设计】

主题: [想讨论什么]

背景: [为什么需要这个？]

我不确定的地方:
  - [不确定1]
  - [不确定2]

AI 请回答:
  1. 你怎么看这个方向？
  2. 给我 2-3 个可选方案，各有什么优缺点？
  3. 你推荐哪个？为什么？
```

## 模板四：日常推进

```
【今日协作】

当前状态: [从 session-handoff.md 复制]

今天要做: [具体做什么]

约束:
  - 一次一个文件
  - 代码前先出设计卡片
  - 每次交付带检查报告

AI 请回答: "准备就绪，确认我们从哪里开始?"
```

---

## AI 必须遵守的回应格式

```
【AI 分析】
[用自己的话复述需求——确认真的懂了]

【AI 方案】
[具体方案：涉及文件、架构层级、依赖——文字描述，不是代码]

【AI 建议】
💡 [专业建议：更好的做法、隐患提醒、边界情况——必须有]

【下一步】
📋 [建议下一步做什么——必须有]
```

## 新 AI 会话开场白（复制使用）

```
你好。我在开发 ZCritical 的 Flutter 项目。

先读这个文件: softwaer/zcritical/.kiro/steering/START-HERE.md

协作规则:
  1. 代码前先出设计卡片
  2. 每次只交付一个文件
  3. 每次交付带检查报告（✅/⚠️/💡/📋）
  4. 你是思考者——主动质疑需求、主动提建议

今天要做: [TODO]
```

## 常见沟通陷阱

| 陷阱 | 怎么解决 |
|------|---------|
| AI 给了 500 行代码，看不懂 | "太多，拆开，一次一个文件" |
| AI 说"这个很简单" | "解释一下为什么这样做" |
| 你不同意 AI 方案 | "给我 2 个替代方案" |
| 你说"不对"但 AI 不改 | 用模板二：现象 + 期望 + 复现步骤 |
| AI 全盘接受需求 | "这个需求有没有隐患？有没有更好的方式？" |
| AI 写完就走 | "告诉我接下来应该做什么" |
